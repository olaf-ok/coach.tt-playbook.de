# Design — i18n DE/EN/ES mit Paraglide + lokalisierte Schlagart-Notation

**Datum:** 2026-04-16
**Projekt:** TT Playbook Trainer
**Status:** Spec — Implementation steht aus

## Motivation

Der Hauptmarkt ist trilingual:

- **Deutsch** — Heimatmarkt (DTTB-/WTTV-Trainer)
- **Spanisch** — Costa Rica (Wohnort des Entwicklers, FECOTEME-Trainer + LATAM)
- **Englisch** — internationale Reichweite

Die App ist heute komplett auf Deutsch hartkodiert. Die Schlagart-Tags (`AS`, `VH-TS`, `RH-TS`, `Schupf`, `Flip`, `VH-Konter`) liegen als deutsche Strings sowohl im Code (`DEFAULT_STROKE_TYPES`) als auch direkt als Speicher-Schlüssel in der IndexedDB (`Stroke.strokeType: 'VH-TS'`).

Recherche zur internationalen Notation (siehe `context/tt-schlagnotation-dreisprachig.md`) zeigt:

- DTTB-Lehrplan-Standard für Deutsch ist klar dokumentiert
- ITTF/USATT/Table Tennis England nutzen `FH`/`BH` durchgängig
- Spanischer Raum mischt englische Fachbegriffe mit echten spanischen Termini (`Saque`, `Devolución`, `Bloqueo`, `Remate`)
- **Eindeutige Empfehlung:** Notation pro Sprache lokalisieren (deutsche Kürzel sind in EN/ES nicht verständlich), interne DB-Codes sprachneutral

Zusätzlich werden neue Features (Auth-Backend, Stripe, Onboarding) sonst weiter hartkodiert. i18n-Lift jetzt: ~110 sichtbare UI-Strings in 22 Komponenten + 10 Stroke-Types + Storage-Migration.

## Scope

### In Scope

1. Paraglide-Setup (Vite-Plugin, `project.inlang/`, Message-Files DE/EN/ES)
2. Sprach-Detection: localStorage-Override → Browser-Sprache → Fallback DE
3. `<html lang>`-Attribut reagiert dynamisch
4. Funktionale `/settings/language`-Seite (System / DE / EN / ES)
5. **Stroke-Type-Refactor:** sprachneutrale interne Codes (`FH_TOPSPIN`, `SERVE`, …) + lokalisierte Labels pro Sprache
6. **Storage-Migration:** bestehende Übungen in IndexedDB von deutschen Strings auf Codes
7. Komplette String-Extraktion aller sichtbaren UI-Strings (Buttons, Labels, Empty-States, Dialoge, aria-labels, placeholders, titles)
8. Übersetzungen DE / EN / ES für alle extrahierten Strings + alle Stroke-Types

### Out of Scope

- **URL-Lokalisierung** (`/de/archive`) — kein SEO-Bedarf, App ist Tool. Spart `reroute`-Hook und Routes-Umbau.
- **Datums-/Zahlenformatierung** mit `Intl.*` — wird gebraucht, sobald Statistiken oder Erstellt-am-Daten angezeigt werden. Nicht heute.
- **ICU-Pluralisierung** — keine pluralen Strings im aktuellen UI.
- **Server-Side i18n** — App läuft im SPA-Mode (`+layout.ts` mit `ssr=false`); Sprache wird rein client-seitig aufgelöst.
- **Custom Stroke-Types** für Pro-User — `StrokeType.isDefault: false` Pfad bleibt unverwendet (war schon vor dieser Spec ungenutzt). Wenn das später kommt, brauchen Custom-Tags ein eigenes Lokalisierungs-Konzept (User schreibt selbst pro Sprache, oder einsprachig).
- **Banana/Strawberry/Chiquita Flick, Lob, Konterblock, Sidespin, Chop, Lob, Smash** — siehe Recherche, sind möglich, aber nicht im aktuellen Default-Set. Wir bleiben beim heutigen 10er-Set, nur lokalisiert.

## Architektur

### Library-Wahl

**Paraglide JS (inlang)** — kompiler-basiert, tree-shakeable, typesafe, SvelteKit-Adapter offiziell. Generiert pro Locale eine `messages/<locale>.json`-Datei und kompiliert sie in `src/lib/paraglide/messages.js` zu typisierten Funktionen. Keine Runtime-Lookups, keine reaktiven Stores nötig — `setLocale()` triggert Reload, App startet mit neuer Locale.

### Verzeichnisstruktur

```
project.inlang/
  settings.json                  # Locale-Liste, baseLocale, plugin
messages/
  de.json                        # Quelle, immer komplett
  en.json
  es.json
src/lib/paraglide/               # GENERIERT — in .gitignore
  messages.js
  runtime.js
src/lib/i18n/
  language-store.svelte.ts       # Wrapper für Mode (system|de|en|es) + localStorage
  stroke-type-labels.ts          # Code → m.* Lookup (UI-Layer)
src/lib/constants/
  strokeTypes.ts                 # NEU: sprachneutrale Codes (Default-Set)
src/lib/db/
  migrations.ts                  # NEU: alte DE-Strings → Codes (Dexie v3)
src/hooks.ts                     # KEIN reroute — kein URL-Prefix
```

### Strategie-Stack

```typescript
// vite.config.ts
paraglideVitePlugin({
  project: './project.inlang',
  outdir: './src/lib/paraglide',
  strategy: ['localStorage', 'preferredLanguage', 'baseLocale']
})
```

**Reihenfolge:**

1. **`localStorage`** — User-Wahl in `/settings/language` gewinnt immer (Paraglide schreibt `localStorage['PARAGLIDE_LOCALE']`).
2. **`preferredLanguage`** — `navigator.languages` (z.B. `['es-CR', 'es', 'en']` → `es`).
3. **`baseLocale`** = `de` als Fallback.

### `project.inlang/settings.json`

```json
{

  "baseLocale": "de",
  "locales": ["de", "en", "es"],
  "modules": [
    "https://cdn.jsdelivr.net/npm/@inlang/plugin-message-format@latest/dist/index.js"

```

### `app.html`

```html
<html lang="%paraglide.lang%" dir="%paraglide.dir%">
```

Paraglide ersetzt Platzhalter beim Build/Dev. Sprach-Wechsel triggert Reload → `lang` neu gesetzt.

### `language-store.svelte.ts`

```typescript
import { browser } from '$app/environment';
import { setLocale, getLocale } from '$lib/paraglide/runtime';

export type LanguageMode = 'system' | 'de' | 'en' | 'es';

const STORAGE_KEY = 'tt-language-mode';

function readMode(): LanguageMode {
  if (!browser) return 'system';
  const v = localStorage.getItem(STORAGE_KEY);
  return v === 'de' || v === 'en' || v === 'es' || v === 'system' ? v : 'system';
}

export const language = $state({
  mode: readMode(),
  current: getLocale() as 'de' | 'en' | 'es'
});

export function setLanguage(mode: LanguageMode): void {
  language.mode = mode;
  localStorage.setItem(STORAGE_KEY, mode);

  if (mode === 'system') {
    localStorage.removeItem('PARAGLIDE_LOCALE');
    location.reload();
  } else {
    setLocale(mode); // Paraglide schreibt + reloadt
  }
}
```

**Trade-off:** `setLocale()` reloadt. Akzeptabel für Settings-Wechsel.

## Stroke-Type-Refactor — Zwei-Schichten-Modell

### Heute (problematisch)

```typescript
// src/lib/constants/strokeTypes.ts
{ shortLabel: 'AS', fullLabel: 'Aufschlag', isDefault: true }
{ shortLabel: 'VH-TS', fullLabel: 'Vorhand-Topspin', isDefault: true }

// src/lib/types/exercise.ts
interface Stroke { strokeType: string | null; }  // 'VH-TS' als Storage-Key

// src/lib/db/seed.ts
{ strokeType: 'VH-TS' }
```

`shortLabel` ist gleichzeitig Anzeige UND Storage-Key UND Filter-Wert im Archive. Sprach-Wechsel wäre datenkorruptiv.

### Neu (Codes intern, Labels lokalisiert)

```typescript
// src/lib/constants/strokeTypes.ts (NEU)
export type StrokeTypeCode =
  | 'SERVE'
  | 'RECEIVE'
  | 'FH_TOPSPIN'
  | 'BH_TOPSPIN'
  | 'FH_BLOCK'
  | 'BH_BLOCK'
  | 'PUSH'
  | 'FLIP'
  | 'FH_COUNTER'
  | 'BH_COUNTER';

export const DEFAULT_STROKE_TYPE_CODES: StrokeTypeCode[] = [
  'SERVE', 'RECEIVE',
  'FH_TOPSPIN', 'BH_TOPSPIN',
  'FH_BLOCK', 'BH_BLOCK',
  'PUSH', 'FLIP',
  'FH_COUNTER', 'BH_COUNTER'
];

export function isStrokeTypeCode(v: string): v is StrokeTypeCode {
  return DEFAULT_STROKE_TYPE_CODES.includes(v as StrokeTypeCode);
}

// src/lib/types/exercise.ts (NEU)
interface Stroke { strokeType: StrokeTypeCode | null; }
```

### Lokalisiertes Label-Lookup

```typescript
// src/lib/i18n/stroke-type-labels.ts
import { m } from '$lib/paraglide/messages';
import type { StrokeTypeCode } from '$lib/constants/strokeTypes';

export interface StrokeTypeLabel {
  code: StrokeTypeCode;
  short: string; // z.B. "VH-T" / "FH-T" / "D-T"
  full: string;  // z.B. "Vorhand-Topspin" / "Forehand Topspin" / "Topspin de derecha"
}

export function strokeTypeLabel(code: StrokeTypeCode): StrokeTypeLabel {
  switch (code) {
    case 'SERVE':      return { code, short: m.stroke_serve_short(),      full: m.stroke_serve_full() };
    case 'RECEIVE':    return { code, short: m.stroke_receive_short(),    full: m.stroke_receive_full() };
    case 'FH_TOPSPIN':    return { code, short: m.stroke_fh_topspin_short(),    full: m.stroke_fh_topspin_full() };
    case 'BH_TOPSPIN':    return { code, short: m.stroke_bh_topspin_short(),    full: m.stroke_bh_topspin_full() };
    case 'FH_BLOCK':   return { code, short: m.stroke_fh_block_short(),   full: m.stroke_fh_block_full() };
    case 'BH_BLOCK':   return { code, short: m.stroke_bh_block_short(),   full: m.stroke_bh_block_full() };
    case 'PUSH':       return { code, short: m.stroke_push_short(),       full: m.stroke_push_full() };
    case 'FLIP':       return { code, short: m.stroke_flip_short(),       full: m.stroke_flip_full() };
    case 'FH_COUNTER': return { code, short: m.stroke_fh_counter_short(), full: m.stroke_fh_counter_full() };
    case 'BH_COUNTER': return { code, short: m.stroke_bh_counter_short(), full: m.stroke_bh_counter_full() };
  }
}
```

### Übersetzungstabelle (Quell-Werte für `messages/*.json`)

Basierend auf der Recherche, mit DTTB-Standard im Deutschen (kürzer als heute, z.B. `VH-T` statt `VH-TS`).

| Code | DE-short | DE-full | EN-short | EN-full | ES-short | ES-full |
|------|----------|---------|----------|---------|----------|---------|
| SERVE | A | Aufschlag | SV | Serve | SAQ | Saque |
| RECEIVE | RS | Rückschlag | REC | Receive | DEV | Devolución |
| FH_TOPSPIN | VH-T | Vorhand-Topspin | FH-T | Forehand Topspin | D-T | Topspin de derecha |
| BH_TOPSPIN | RH-T | Rückhand-Topspin | BH-T | Backhand Topspin | R-T | Topspin de revés |
| FH_BLOCK | VH-B | Vorhand-Block | FH-BLK | Forehand Block | D-BLQ | Bloqueo de derecha |
| BH_BLOCK | RH-B | Rückhand-Block | BH-BLK | Backhand Block | R-BLQ | Bloqueo de revés |
| PUSH | Schupf | Schupf | Push | Push | Push | Push |
| FLIP | Flip | Flip | Flip | Flip | Flip | Flip |
| FH_COUNTER | VH-K | Vorhand-Konter | FH-CTR | Forehand Counter | D-CONT | Contraataque derecha |
| BH_COUNTER | RH-K | Rückhand-Konter | BH-CTR | Backhand Counter | R-CONT | Contraataque revés |

**Bewusste Entscheidungen (mit User abgestimmt 2026-04-16):**

- DE: DTTB-Standard `VH-T`/`RH-T` (kürzer als heute `VH-TS`/`RH-TS`).
- DE/EN/ES: `Schupf` / `Push` und `Flip` werden ausgeschrieben (statt DTTB-`S` / DTTB-`F`) — kurz genug, lesbarer auf Buttons.
- ES: alle paarigen Kürzel mit `D-`/`R-`-Prefix (Derecha/Revés) und spanischer Schlagart (`Topspin`, `Bloqueo`, `Contraataque`). Trainer-Feedback aus Costa Rica: `Topspin` ist gebräuchlicher als `Loop`.
- EN: konsistent `T` für Topspin (`FH-T`/`BH-T`), nicht `L`/`Loop` — Konsistenz quer durch DE/EN/ES.

### Komponenten-Anpassungen

**`StrokeTypeButtons.svelte`** (heute liest direkt `DEFAULT_STROKE_TYPES`):

```svelte
<script lang="ts">
  import { DEFAULT_STROKE_TYPE_CODES, type StrokeTypeCode } from '$lib/constants/strokeTypes';
  import { strokeTypeLabel } from '$lib/i18n/stroke-type-labels';

  interface Props {
    activeType: StrokeTypeCode | null;
    onSelect: (code: StrokeTypeCode) => void;
  }
  let { activeType, onSelect }: Props = $props();
</script>

{#each DEFAULT_STROKE_TYPE_CODES as code}
  {@const label = strokeTypeLabel(code)}
  <button
    type="button"
    class:active={activeType === code}
    title={label.full}
    onclick={() => onSelect(code)}
  >
    {label.short}
  </button>
{/each}
```

**`Archive.svelte`** Filter-Chips: lesen `DEFAULT_STROKE_TYPE_CODES` und zeigen `strokeTypeLabel(code).short`. Vergleich der Übungen geschieht über `code === stroke.strokeType` (sprachneutral).

**`StepsPanel.svelte`** zeigt den aktiven Tag — wenn `stroke.strokeType` ein gültiger Code ist, `strokeTypeLabel(...).short` rendern.

### Storage-Migration

Bestehende Übungen in IndexedDB haben `strokeType: 'VH-TS' | 'AS' | 'VH-Konter' | …`. Beim Schema-Upgrade auf v3 mappen wir die alten DE-Strings auf neue Codes.

```typescript
// src/lib/db/migrations.ts
const LEGACY_DE_TO_CODE: Record<string, StrokeTypeCode> = {
  'AS': 'SERVE',
  'RS': 'RECEIVE',
  'VH-TS': 'FH_TOPSPIN',
  'RH-TS': 'BH_TOPSPIN',
  'VH-Block': 'FH_BLOCK',
  'RH-Block': 'BH_BLOCK',
  'VH-BL': 'FH_BLOCK',     // Tippfehler aus seed.ts — sicherheitshalber
  'Schupf': 'PUSH',
  'Flip': 'FLIP',
  'VH-Konter': 'FH_COUNTER',
  'RH-Konter': 'BH_COUNTER'
};

export function migrateStrokeType(legacy: string | null): StrokeTypeCode | null {
  if (!legacy) return null;
  if (isStrokeTypeCode(legacy)) return legacy; // schon migriert
  return LEGACY_DE_TO_CODE[legacy] ?? null;    // unbekannt → leer (Trainer kann neu zuweisen)
}
```

**Dexie-Migration v2 → v3:**

```typescript
// src/lib/db/database.ts
db.version(3).stores({
  exercises: 'id, name, createdAt, updatedAt',
  playlists: 'id, name, createdAt, updatedAt'
}).upgrade(async tx => {
  await tx.table('exercises').toCollection().modify(ex => {
    ex.strokes = ex.strokes.map((s: any) => ({
      ...s,
      strokeType: migrateStrokeType(s.strokeType)
    }));
  });
});
```

`seed.ts` wird angepasst: `'VH-TS'` → `'FH_TOPSPIN'`, `'AS'` → `'SERVE'`, `'VH-BL'` → `'FH_BLOCK'`, etc.

## Build / Tooling

### Paraglide initialisieren

```bash
cd ~/Developer/tt-playbook-trainer
npx @inlang/paraglide-js init
```

CLI fragt: framework (sveltekit), locales (de,en,es), baseLocale (de). Generiert `project.inlang/`, `messages/`, fügt Vite-Plugin in `vite.config.ts` ein, `.gitignore`-Eintrag für `src/lib/paraglide/`.

**Falls die CLI eine andere Strategy-Default setzt** (z.B. `['url']`), manuell auf `['localStorage', 'preferredLanguage', 'baseLocale']` ändern.

### svelte-check + Build

`m.*`-Funktionen sind voll typisiert. svelte-check meldet, wenn ein Key in einer Sprache fehlt. Der Stroke-Type-Refactor verändert die `Stroke.strokeType`-Signatur (`string | null` → `StrokeTypeCode | null`), TypeScript fängt alle Verwender.
